Angular is a framework that let's you build out web apps really fast! It's built using TypeScript. What's the difference between AngularJS and Angular? Let's briefly cover the 8 main building blocks of Angular...

Modules

Modules are Angular's way of organizing and cleanly decoupling certain workflows or application domains. Angular apps can be viewed as a module tree, with a top-level module. Most will typically have a core module and shared module as well. Larger apps will typically have multiple feature modules as well. For more information, check out the ngModules Angular Documentation

Components

A component controls a patch of screen called a view. You define a component's application logic (what it does to support the view) inside a class. When you generate a component through the CLI, it creates the following files:

• Component • Template • Spec • Stylesheet Templates

You define what a component displays with its corresponding template. It's nothing more than an HTML file spiced up with custom directives.

 Directives

Angular templates are dynamic. When Angular renders them, directives tell it how to transfer the DOM. Components are technically directives. Two other kinds of directives exist: structural and directives. They tend to appear within an element tag as attributes do, sometimes by name but more often as the target of an assignment or a binding. *ngFor is a structural directive.

Structural directives alter layout by adding, removing, and replacing elements in DOM.

Attribute directives alter the appearance or behavior of an existing element. In templates they look like regular HTML attributes, hence the name. ngModel is an attribute directive.

 Services

Service is a broad category encompassing any value, function, or feature that your application needs. Almost anything can be a service. A service is typically a class with a narrow, well-defined purpose. It should do something specific and do it well. Components consume services. Services are singletons. Some examples include: • logging service • back-end service • currency conversion calculator • application configuration The Angular CLI makes it really easy to use the command line to quickly generate code following Angular best practices. You can use it to quickly accomplish the following types of activities: • create an new application that already works • generate components, modules, services, and pipes • serve your app locally while developing • build an optimized, production version of your app • run tests, lint your code, and auto-format

Angular Material is a library of out-of-the-box material design UI components built to integrate seamlessly with Angular.

• Form controls (inputs, selects, autocomplete, datepickers...) • Layout components (cards, grids, tabs...) • Buttons & indicators (buttons, tooltips, chips, spinners...) • and more...

The Redux JavaScript library was made famous by React and Elm, but its design patterns can also be applied in Angular. The general idea is that all application data is kept in a single JavaScript object on the client, giving you a predictable tree of state changes.

1. All application data is one single object known as a store, representing the application’s state. It's essentially a mini client-side database that holds all data being consumed by an app at any given point in time.

2. A state object cannot be changed - it is immutable.

3. When data changes, the existing state is duplicated, then a new object is created with the updates. In Angular this data is treated as an RxJS Observable, allowing us to subscribe to it from anywhere in the app.

4. State can only be changed via an action, which is also an object. It includes a type (the action name) and an optional payload (the action data), for example {{'{'}}type: 'DELETE_ITEM', payload: 123 {{'}'}}.

5. When an event is emitted, for example a button click, the action is sent to a reducer function to converts the old state into the new state.
